2. 키 정의 => RDB 인 이유가 데이터들이 관계를 맺고 있기 때문
1)정의
 - 키는 특정행은 식별할 때 사용하는 속성
 - 키는 중복을 허용하지 않음

2) 기본키
 - 테이블(릴레이션)의 대표키 <각행의 항목명 인 것 같아..줄 알았더니 아니고 그 중에 겹치면 안되는 진짜 중요한 애들>
   : 테이블 내에서 하나를 선정하여 대표로 삼는 키 
 - 생성방법
   CREATE TABLE 테이블명
   (컬럼명1 타입(길이),
    컬럼명2 타입(길이)
    
    constraints [pk명] primary key (기본키1, 기본키2..));

3) 외래키
 - 다른 테이블(릴레이션)의 기본키를 참조하는 속성	
 - 무결성 참조 에러가 날 수 있어
  <외래키에 담긴 특정 값만 수정하면 문제가 발생할 수 있어..>
 - 생성방법
   CREATE TABLE KOPO_PRODUCT_VOLUME
   (컬럼명1 타입(길이),
    컬럼명2 타입(길이)
    
    constraints [pk명] foreign key(컬럼명) references 기본키포함테이블명(해당테이블 기본키))       
     
   ^이런 컨셉을 모르면 문제가 발생할 수 있어. 
   삭제를 할 때도 관계를 고려하면서 해야하니깐 



3. 무결성 제약조건 정의
 - 데이터 무결성: 데이터베이스에 저장된 데이터의 일관성과 정확성을 지키는 것

 -도메인 무결성 제약조건: 지정된 컬럼 속성 값 외에 허용하지 않음
  <숫자 들어가기로 한 자리에 문자 들어가면 에러가 발생함>

 -개체 무결성 제약조건: 기본키 제약 
  ^만약 기본키가 하나도 없다면, 아직 정렬이 되지 않은 테이블로 의심해라

 -참조무결성 제약조건: 외래키 제약
  ^참조하고 있을 때는 내껄 지우고 참조하는 데이터 지우고, 
   걔한테 새로운 정보가 있으면 내꺼에도 새롭게 넣어 <똑같이 하나봐>
